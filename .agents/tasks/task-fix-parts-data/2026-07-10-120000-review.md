# Beyblade X parts data correction from wiki sources

This change rewrites the parts seed data in `prisma/data/meta-parts.ts` to align with authoritative Beyblade Wiki Fandom data. The core work is threefold: fix manufacturer misattributions (Wizard Arrow, 4-60, Point moved from HASBRO to TAKARA_TOMY), drastically narrow the Hasbro-exclusive list to only blades with N/A for Takara Tomy names, and expand coverage with new blades, ratchets, and CX parts from the wiki. The ratchet-integrated blade category is fleshed out with both TT and Hasbro entries, and the series-inference Sets are updated to match.

**Watch for:** Glory Valkyrie from `wikiBladeNames` lacks `ratchetIntegration: BLADE` despite being a ratchet-integrated UX_EXPANDED blade — the wiki-generated entry gets `NONE` because only hand-curated entries set that field (confirmed). "Hover Wyvern" and "Wyvern Hover" coexist as separate database entries for the same physical blade under different naming conventions (confirmed). The removal of several ratchets from `wikiRatchetNames` is safe only because they already exist in `cxRatchetNames` — but this creates an implicit dependency between two arrays that look independent (confirmed).

## High-level view

The Hasbro-exclusive list was previously inflated: X-Over Project blades, BX_EXPANDED blades, and Hasbro-debut blades with valid TT names were all lumped in. The fix correctly reclassifies them — X-Over and Hasbro-debut blades move into `wikiBladeNames` (TAKARA_TOMY manufacturer), and only four blades that genuinely have no TT equivalent remain in `wikiHasbroOnlyBladeNames`.

The series-inference Sets grew to handle the new blade roster. The new ratchet-integrated entries in the hand-curated `metaParts` array correctly set both `series` and `ratchetIntegration`, except that the wiki-generated Glory Valkyrie entry survives dedup as a separate slug and misses the ratchet flag.

The ratchet list cleanup moves CX-exclusive ratchets out of `wikiRatchetNames` into `cxRatchetNames`, adding `3-85` and `5-60`. The CX line gains three lock chips (Bucks, Tiga, Valkyrie), two metal blades (Hurricane, Tread), and two over blades (I, T) — but "Stag" remains alongside "Bucks" for what may be the same physical chip.

## Hasbro-exclusive reclassification

The old `wikiHasbroOnlyBladeNames` contained 16 entries including X-Over Project blades (DranzerSpiral, DrigerSlash, etc.), BX_EXPANDED blades (CyclopsEye, DranStrike, HeavensRing, SiegSuperion), and Hasbro-debut blades with valid TT names (BatGust, TriceraSpiky). All of these had both Hasbro and Takara Tomy names on the wiki, meaning they were incorrectly flagged as Hasbro-exclusive.

The new list contains only four entries: Clamp Crab, Yell Kong (Basic Line), Hack Viking, and Stun Medusa (Unique Line). The series inference works for Hack Viking and Stun Medusa because they appear in `uxBladeNames` — the manufacturer comes from which array generates the part, while the series comes from which Set the resulting name matches during `applyMetaPartMetadata`. This decoupling is non-obvious but correct.

## Ratchet-integration gap on Glory Valkyrie

`wikiBladeNames` includes `GloryValkyrie`, which generates a part with slug `glory-valkyrie`, manufacturer TAKARA_TOMY, and series UX_EXPANDED (because `uxExpandedBladeNames` contains "Glory Valkyrie"). However, `applyMetaPartMetadata` does not infer ratchet integration from set membership — it only preserves what's already on the part or defaults to `NONE`.

The hand-curated "Glory Valk" entry (slug `glory-valk`) does not collide on slug, so both survive dedup. The result is two entries for what is likely the same blade: one with ratchet integration, one without. BulletGriffon and HellsNether avoid this problem because their wiki-generated slugs match the hand-curated slugs exactly, so dedup keeps the hand-curated version with `ratchetIntegration: BLADE`.

## Hover Wyvern naming collision

The pre-existing hand-curated "Hover Wyvern" (slug `hover-wyvern`, HASBRO, BX, S-tier with weight data) and the new wiki-generated "Wyvern Hover" (slug `wyvern-hover`, TAKARA_TOMY, UX) represent the same physical blade under Hasbro vs. Takara Tomy naming conventions. The FEAT-001 findings acknowledge this was intentional.

The seed script upserts by slug, so both entries will exist in the database. A user with the physical blade would see two catalog entries with different series assignments (BX vs UX) and different manufacturers. The series discrepancy is particularly confusing: the Hasbro version gets BX (default fallback, since "Hover Wyvern" isn't in any series Set) while the TT version gets UX (because "Wyvern Hover" is in `uxBladeNames`).

## Ratchet list restructuring

The old `wikiRatchetNames` contained 34 entries including seven that belonged exclusively to the CX line. The new version removes those seven (they're covered by `cxRatchetNames`) and adds `3-85` and `5-60`. No ratchets are lost in the combined output.

The addition of `5-60` to `wikiRatchetNames` is redundant with the hand-curated entry in `metaParts` (which carries weight and tier data). The dedup silently keeps the hand-curated version. A future editor changing the wiki-generated `5-60` entry won't understand why their change has no effect.

## CX line: Stag and Bucks coexistence

"Stag" remains in `cxLockChipNames` alongside the newly added "Bucks", even though the FEAT-001 notes indicate Bucks is the TT name for Stag. The database will contain both `cx-lock-chip-stag` and `cx-lock-chip-bucks` as separate entries. If these are the same physical chip, one should be removed or they should be linked via notes.
